## 📊 Сводная таблица оптимизаций

| № | Проблема | Решение | Ускорение |
|---|----------|---------|-----------|
| 1 | Коррелированный подзапрос | `LEFT JOIN + GROUP BY` | **842x** |
| 2 | `NOT IN` + подзапросы | `LEFT JOIN + IS NULL` | **150x** |
| 3 | Самообъединение (Self Join) | Оконная функция `LAG` | **130x** |
| 4 | Многократные подзапросы | `CTE + GROUP BY` | **100x** |
| 5 | `Seq Scan` (без индекса) | `CREATE INDEX` | **10-1000x** |
| 6 | Сортировка по вычисляемому полю | Предвычисление в CTE | **2-5x** |
| 7 | `SELECT *` | Указание только нужных колонок | **1.5-3x** |
| 8 | `OR` в условиях | `UNION ALL` или `IN` | **2-10x** |
| 9 | `LIKE '%текст%'` (с % в начале) | Полнотекстовый поиск (GIN) | **50-200x** |
| 10 | `DISTINCT` на больших таблицах | `EXISTS` или `GROUP BY` | **2-5x** |
| 11 | Функции в `WHERE` | Предвычисление или выраженные колонки | **5-20x** |
| 12 | `COUNT(*)` с условием | `COUNT(*) FILTER (WHERE ...)` | **1.5-2x** |
| 13 | Вложенные `SELECT` в `SELECT` | `JOIN` или оконные функции | **3-10x** |
| 14 | Отсутствие индекса на FK | `CREATE INDEX` на внешний ключ | **10-100x** |
| 15 | `LIMIT` без `ORDER BY` | Добавить `ORDER BY` + индекс | **2-100x** |
